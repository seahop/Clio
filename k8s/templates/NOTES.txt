       kubectl get certificate -n {{ .Values.namespace }}

MANAGING SECRETS
  Credentials are stored in the 'clio-secrets' Secret.  To rotate a value:
    kubectl patch secret -n {{ .Values.namespace }} clio-secrets \
      --type=merge -p '{"stringData":{"REDIS_PASSWORD":"new-value"}}'
  Then roll the backend deployment:
    kubectl rollout restart deployment -n {{ .Values.namespace }} \
      {{ include "clio.fullname" . }}-backend

UPGRADING
  helm upgrade clio ./k8s -n {{ .Values.namespace }} -f my-values.yaml

  Image tag pinning (recommended for production):
    helm upgrade clio ./k8s -n {{ .Values.namespace }} \
      --set backend.image.tag=v1.2.3 \
      --set frontend.image.tag=v1.2.3
